# Overview
ReFlexHire는 전통적인 헤드헌팅의 고비용·저효율 구조를 혁신하고, 채용 이후의 온보딩 경험까지 아우르는 **경험 기반 채용 플랫폼**입니다.
기존 채용이 ‘사람의 연결’에 머물렀다면, ReFlexHire는 ‘사람의 여정’을 함께 설계합니다.

대상 고객은 빠르게 인재를 채용하고자 하는 스타트업부터 브랜딩을 중시하는 중견·대기업까지 포함되며, 채용 이후 구성원의 정착과 조직 문화 적응까지 지원하고자 하는 조직입니다.

채용 수수료를 정액화(연봉 대비 10%)하고, 그 비용을 기반으로 웰컴키트, 케이터링, 문화 적응 콘텐츠 등 **입사자의 경험 설계에 투자**함으로써, 단순 채용을 **브랜드 자산**으로 전환할 수 있습니다.

# Core Features
1. **정액 수수료 기반 채용 서비스**
   - 연봉과 무관하게 10% 정액 수수료를 채택하여 예측 가능한 비용 구조 제공
   - 기업은 연봉 구간별로 필수 온보딩 패키지를 연동 구매하게 되며, 이는 입사자의 만족도와 조직 정착률을 높이기 위한 직접적인 투자로 작용함

2. **온보딩 경험 패키지 시스템**
   - 웰컴키트, 커피차, 입사자 맞춤 콘텐츠, 동료 간 식사 등으로 구성된 다양한 패키지를 연봉 구간별로 자동 연동
   - 서비스는 ReFlexHire 플랫폼 내 파트너 마켓플레이스를 통해 자동 매칭

3. **ReFlex_OnTheGo (이동형 채용 경험 서비스)**
   - 프라이빗 리무진 기반 인터뷰 공간 제공
   - 핵심 인재, 임원 채용 시 브랜드 중심 경험 제공 (브랜드 이미지 향상 + 채용 민감도 완화)

4. **고객용/리크루터용 대시보드**
   - 채용 진행, 패키지 선택, 정산까지 모두 투명하게 관리 가능
   - 리크루터는 자신이 추천한 인재의 현황과 실적을 실시간 확인 가능

5. **온보딩 크레딧 시스템**
   - 수수료 중 일부를 온보딩 크레딧으로 환산하여 입사자에게 사용할 수 있게 함
   - 패키지 선택, 일정 조율, 배송까지 자동화된 워크플로우로 연결

# User Experience
## 사용자 페르소나
1. **스타트업 HR 매니저 "Jenny"**
   - 빠르게 채용하되 구성원 경험을 놓치고 싶지 않음
   - 채용만 해주던 플랫폼보다, 입사까지 챙겨주는 서비스에 매력 느낌

2. **중견기업 인사팀장 "David"**
   - 기존 에이전시와는 차별화된 브랜딩/온보딩 서비스가 필요
   - 내부 조직 설득에 도움이 되는 "입사 첫인상 경험" 제공

3. **프리랜서 리크루터 "Soojin"**
   - 경쟁 에이전시 대비 차별화된 제안이 필요
   - 채용 + 온보딩 번들링을 통해 클라이언트 설득력 향상

## 주요 유저 플로우
1. **기업 - 채용 요청 등록 → 채용 → 온보딩 패키지 선택 → 배송 및 완료 확인**
2. **리크루터 - 채용 요청 수신 → 인재 추천 → 채용 완료 → 수수료 수령**
3. **ReFlexHire 관리자 - 거래 관리 → 벤더 연결 및 진행 현황 확인**

## UI/UX 고려사항
- 직관적인 대시보드 구성 (클라이언트/리크루터용 이중 인터페이스)
- 온보딩 크레딧 시각화 및 패키지 커스터마이징 지원
- 반응형 웹 (모바일·태블릿 지원)
- 다국어 및 현지화 확장성 고려

# Technical Architecture
- **Frontend**
  - Next.js (React 기반 SSR/CSR 혼합 구조)
  - Client/Recruiter/Admin 포털 분리

- **Backend**
  - Node.js + Express.js 또는 Django 기반 REST API
  - OAuth2/JWT 인증, RBAC 권한 체계 구축
  - PostgreSQL 또는 MongoDB

- **Core Modules**
  - 채용 요청 등록 및 매칭 엔진
  - 온보딩 패키지 빌더 및 마켓플레이스
  - 결제/인보이싱 모듈 (Stripe 또는 Toss 연동)
  - 벤더 API Hook (주문 상태 및 배송 연동)

- **Infrastructure**
  - AWS 기반 (EC2, RDS, S3, Lambda), 또는 Vercel + Firebase
  - 보안: HTTPS, WAF, IAM Role, 백업 전략 포함

# Risks and Mitigations
- **리스크**: 벤더 배송 지연, HR팀의 기술 적응 곤란, 초기 고객 확보 난항
- **해결방안**:
  - 수동 주문 시스템 및 관리자 개입 구조 유지
  - 미리 설정된 온보딩 패키지 템플릿 제공
  - 스타트업 타깃 고객에 무료 베타 제공 → 후기 기반 초기 레퍼런스 확보
